# 流式 TTS：导入即听

## 核心原则

**导入即听，生成追着播放跑。** 用户在导入阶段最多等 30 秒（前 5 个场景的 TTS），之后所有音频生成都在后台追赶用户的播放速度。

单人说书模式（当前版本），不做有声剧。有声剧是后续版本的事。

## 导入流程

```
拖入文件 → 解析分章 → 规则切场景 → 生成前 5 个场景 → 标记"可听"
                                              ↕
                                       约 20-30 秒
```

- 导入后回书架，不跳转阅读页
- 书架卡片显示场景生成进度
- 用户随时点开听

## 场景切分

单人说书模式去掉 LLM 导演环节，改用纯规则切分：

- 按段落边界断开
- 单段超过 500 字时在句号处再切一刀
- 全部使用统一声音 + "沉稳"情绪标签
- 切分在导入时完成，耗时忽略不计

## 生成策略

| 优先级 | 触发条件 | 行为 |
|--------|----------|------|
| P0 紧急 | 当前场景未生成 | 立即单段最高优生成 |
| P1 当前章 | 正在播放的章节 | 保持领先用户 10 个场景 |
| P2 下一章 | 当前章播放过半 | 预生成下一章全部场景 |
| P3 书架预热 | 系统空闲 | 最近阅读的书第 1 章前 5 段 |

并发数保持 5。

## 场景时间线（替代进度条）

PlayerBar 改为场景导航：

```
段 3 / 1247    ● ● ● ◉ ●● ○ ○ ○ ○ ○ ○ ○ ...
               已播    当前 已就绪  等待/生成中
```

| 状态 | 视觉 | 交互 |
|------|------|------|
| 已播放 | 实心灰色点 | 点击跳转 |
| 当前播放 | 高亮色 + 微光 | — |
| 已生成待播 | 实心浅色点 | 点击跳转 |
| 生成中 | 虚线闪烁 | 不可点击 |
| 等待 | 空心点 | 不可点击 |

- 时间线可横向滑动
- 主要显示「段 N / 总数」，时间降级为次要信息

## 书架卡片

```
┌──────────────┐
│   龙族        │
│   58 章       │
│   ━━━━━━ 45%  │  ← 场景生成进度
│   562/1247 段  │
└──────────────┘
```

## 前端改动

1. **ImportPage**：导入后不跳转，展示预处理进度面板；前 5 段生成完显示"开始阅读"按钮
2. **PlayerBar → SceneTimeline**：水平场景点导航，替代传统进度条
3. **BookCard**：增加场景生成进度条
4. **ReaderPage**：适配场景时间线，去掉轮询等待逻辑（场景已预就绪）

## 后端改动

1. **services.ts**：
   - `generateChapterAudio()`：LLM 导演替换为规则切分，FIRST_BATCH_SIZE 从 10 改为 5
   - `importBook()`：导入后同步切分场景 + 触发前 5 段 TTS + 后台队列生成剩余
   - 新增场景级生成队列和优先级调度
   - 新增 `getBookGenerationProgress(bookId)` 查询全书生进度
2. **tts/route.ts**：不变（POST 触发、GET 查询状态）
3. **books API**：导入完成后不再只返回 book 数据，附带场景生成状态

## 不做的事

- 不实现有声剧模式（多角色、LLM 导演）——这是后续版本
- 不改变 TTS API 调用方式（仍用 MiMo v2.5-tts）
- 不引入新的存储引擎或消息队列
- 不改变现有的 3GB 缓存限制策略
